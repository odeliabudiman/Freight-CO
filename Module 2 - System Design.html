③ Processing & Business Logic
Core Logic — Sprint 1
⚙️
Delay Calculator
Lambda / Supabase Edge Fn
Computes delay_minutes = actual_time − scheduled_time. Updates delivery record. Flags status as at_risk if in_transit AND current_time > scheduled_time + 30 min.
Rule: +45 min = alert
🚨
Alert Engine
n8n / Make.com Workflow
Triggered when delay_minutes ≥ 45. Looks up hub coordinator contact. Fires SMS via Twilio + email via SendGrid. Deduplicates — one alert per delivery per threshold breach.
Sprint 1 · Core Feature
🕐
Scheduled Poller
Cron Lambda (15 min)
Scans in_transit deliveries where no actual_time received. Re-checks against current time. Catches silent failures — drivers who never updated status.

④ Storage · Notifications · Dashboard
Persistence & Output — Sprint 1 + 2
🗄️
Delivery Database
Supabase (PostgreSQL)
Stores all deliveries with delivery_id, driver, hub, times, status, delay_minutes. Row-level security. Real-time subscriptions for dashboard. Free tier sufficient for MVP.
Managed Postgres
📱
SMS Alert
Twilio
Sends formatted alert to hub coordinator: driver name, hub, delivery ID, minutes late. Pay-per-message — no fixed infra cost. Escalation to ops manager at +90 min.
Sprint 1
📧
Email Alert
SendGrid
Richer email with delivery detail, driver history, and suggested action link. Same trigger as SMS. HTML template stored in n8n/Make workflow. Free tier: 100/day.
Sprint 1
📊
Ops Dashboard
Retool / Metabase
Real-time table of all in-transit deliveries. KPIs: on-time %, avg delay by hub/driver. Filter by hub. No-code setup — connects directly to Supabase. Hub coordinators can self-serve.
